03 / Math / Algorithm Used

Math / Algorithm Used

The review layer uses pattern matching and cross-referencing rather than numerical algorithms:

  • TBD detection: Regex r"\[(TBD|PLACEHOLDER|INSERT|SOURCE|EVIDENCE)[^\]]*\]" with case-insensitive matching. Tested against 5 marker variants.
  • GHG boundary completeness: Keyword presence check in Section 3.3 text for 5 required elements: included sources, excluded sources, geographic boundary, temporal boundary, project type.
  • Monitoring parameter coverage: String inclusion check comparing section 5.1/5.2 combined text against 4 ACM0022 required parameters (waste throughput, biogas CH4%, recovered electricity, grid emission factor).

06 / Limitations / Second-Best Alternative

Limitations / Second-Best Alternative

  • Keyword-based boundary check: The GHG boundary completeness check looks for keyword presence, not semantic understanding. A section could mention "excluded" in a different context and pass.
  • Monitoring parameter coverage is methodology-specific: Currently hardcoded for ACM0022. Adding ACM0003 or other methodologies requires extending the parameter list.
  • TBD regex could miss creative placeholder formats: Markers like "<TODO>" or "{{FILL}}" are not detected. The regex is intentionally narrow to avoid false positives.
  • Second-best alternative: Instead of adding checks to consistency.py, we could have built a separate "completeness engine" that validates against the schema using a more formal DSL. Rejected because it would add significant complexity for marginal gain over keyword checks.
07 / Errors / Warnings / Flags

Errors / Warnings / Flags Encountered

  • DraftSection required fields (fixed): Initial tests failed because DraftSection.__init__ requires confidence, provenance, issues, provider. Fixed by adding a test helper _make_section() that supplies defaults.
  • Section key format mismatch (fixed): TBDTracker initially produced section labels like "1.1.10" because it concatenated section_id + sub_section_id. Fixed to detect when sub_section_id already includes the parent ID and use it directly.
  • DOCX renderer key mismatch (fixed): render_tbd_appendix expected keys "section" and "suggested_evidence" but TBDReport.to_dict() emits "section_id", "sub_section_id", and "evidence_type". Fixed renderer to match actual output format.
  • No regressions: All 204 existing + new tests pass.
