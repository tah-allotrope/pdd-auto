Incidental Fix

The new test surfaced a real bug in RetrievalIndex.stats(): it called cur.fetchone() twice per metric (total = cur.fetchone()[0] if cur.fetchone() else 0), consuming the row in the condition and dereferencing None. Rewrote both metrics to fetch once. Root-cause fix, not a test workaround.
