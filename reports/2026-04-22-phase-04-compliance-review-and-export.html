Number extraction: re.findall(r"\d[\d,]*", text)[0].replace(",", "") — captures the first comma-formatted integer in section text. Handles "50,000 tCO2e/year" → 50000.

State machine: drafted → needs-input | needs-domain-review | ready-for-human-edit → approved. Direct drafted → approved is blocked. Reversion allowed: needs-input → drafted.

06 / Limitations / Second-Best Alternative

Limitations / Second-Best Alternative

  • DOCX requires python-docx (not yet installed): pip install python-docx is required before pdd-agent export will produce output. Gracefully degrades with an error log rather than crashing.
  • Real LLM not wired: Only NoopProvider is operational. All sections produce structured placeholders. OpenAIProvider and OllamaProvider stubs exist but are not configured. Until an LLM is wired, the full drafting + review pipeline cannot be exercised end-to-end.
  • Double-counting guard DC-01 string matching: The credit allocation check looks for the substring "credit allocation" in §1.16. A more robust approach would parse a structured table. Current approach is a pragmatic heuristic — domain expert review is mandatory for dual-claim projects.
  • Number extraction from prose: _extract_first_number uses regex on raw draft text. If section text contains multiple numbers (e.g., "capacity is 25 MW and annual waste is 100,000 tonnes"), the first match may not be the intended value. This is why HIGH/CRITICAL sections require human review before finalization.
  • 4 malformed .norm.json files: Yanjiang, Linfen, Shunping, Yingoku have JSON parse errors. Fixing them would increase corpus coverage from 9/13 to 13/13. These are excluded from FTS5 indexing, reducing retrieval recall for those projects.
07 / Errors / Warnings / Flags

Errors / Warnings / Flags Encountered

  • Test failure: test_blocks_critical_severity — ReviewCheckResult.passed started as True but wasn't updated when a CRITICAL flag was added. Fixed: add_check() now sets self.passed = False when a CRITICAL flag is appended.
  • Test failure: test_matching_net_values_passes — Number extraction regex [\d,]+\.?\d* split "50,000" into "50" and "000" as separate matches. Fixed: changed to \d[\d,]* which captures the full formatted integer as one unit.
  • Test failure: test_invalid_transition_drafted_to_approved_blocked — YAML allowed direct drafted → approved transition, contradicting the domain rule that every section must go through ready-for-human-edit before final approval. Fixed: removed approved from drafted's can_transition_to set in both the YAML and _TRANSITIONS dict.
  • Test failure: test_all_approved_true_when_everything_approved — Same root cause as above: the two-step path (drafted → ready-for-human-edit → approved) was not reflected in the test. Fixed: test now calls set_state(READY_FOR_HUMAN_EDIT) before set_state(APPROVED).
  • Import error: run_post_draft_checks not found — Imported as a standalone function but it's a method on MethodologyRules. Fixed import to use get_methodology_rules() instance and call rules_engine.run_post_draft_checks(section_texts).
  • CLI build-index corpus-dir mismatch: The CLI argument is --corpus-dir but RetrievalIndex.build(normalized_dir=) parameter name is normalized_dir. Fixed call: idx.build(normalized_dir=Path(args.corpus_dir)).
  • Deprecation warning: datetime.utcnow() — Used in section_orchestrator.py:61 (existing, PHASE-03). Marked as known issue. Should be replaced with datetime.now(datetime.UTC).
