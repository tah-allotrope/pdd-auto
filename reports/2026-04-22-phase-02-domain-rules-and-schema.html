06 / Limitations / Second-Best Alternative

Limitations / Second-Best Alternative

  • Corpus not live-parsed in tests — schema unit tests use the schema file directly; corpus parse tests skip when data/corpus/normalized/ is absent. Corpus integration tests (7 skipped) run only when the full ingest pipeline has been executed.
  • Heading alias matching is substring-based — approximate matching is a pragmatic fallback for PDF headings that deviate from canonical form. It could misclassify in rare cases; manual review gates on HIGH/CRITICAL sections mitigate this.
  • post_draft_checks() uses regex on section text — string-pattern matching (e.g., "yes"/"no" markers) is a proxy for structural validation. A full NLP or structured section parser would be more robust but was deferred to keep scope controlled.
  • gws CLI not on PATH for subprocess — The gws binary path is hardcoded as Path.home()/AppData/Roaming/npm/gws.cmd. If npm global path differs on another machine, the Drive wrapper breaks. No dynamic PATH discovery was implemented.
07 / Errors / Warnings / Flags

Errors / Warnings / Flags Encountered

  • FileNotFoundError: schemas — _load_schema() resolved the schema path relative to section_parser.py using Path(__file__), which placed it under C:\Users\tukum\Downloads instead of the project root. Fixed by defaulting to Path(__file__).parent.parent.parent.parent / "schemas", which correctly resolves to the project root regardless of where the module is invoked from.
  • TypeError: make_minimal_input() — The test helper was defined with **overrides kwargs but tests called it with positional string keys like "location.latitude". Fixed by adding a module-level _ov() helper that applies dot-path overrides to a base dict.
  • Negative net emissions test assertion — The validator raised the mismatch error first (net −10,000 vs expected −10,000 was actually correct — the negative net error was triggered second). Fixed by reordering checks: negative check runs before the math check so the right error message is raised.
  • Unicode surrogate characters — Some PDF extractions contained embedded math Unicode surrogates from symbol fonts. Already handled in PHASE-01 normalize.py via errors="surrogatepass".
