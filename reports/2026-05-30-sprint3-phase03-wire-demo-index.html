Design Notes

  • Precedence: production corpus.fts.db always wins; demo.fts.db is a fallback only when the production index is absent. Separate filenames avoid any conflict (RISK-02-01).
  • Singleton coherence: build_demo_index() calls a new set_retrieval_index() so a freshly built index is used immediately in-process, and get_retrieval_index() also resolves demo.fts.db from disk on first use (RISK-02-02 covered both ways).
  • Shared helper: ensure_demo_index() lives in scripts/_demo_helpers.py (used by both scripts), and no-ops when an index already exists or the demo corpus is absent.
