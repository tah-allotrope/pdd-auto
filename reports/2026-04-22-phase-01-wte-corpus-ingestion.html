06 / Limitations / Second-Best Alternative

Limitations / Second-Best Alternative

  • OCR not attempted yet. Four PDFs in an earlier dry run failed with surrogate-character encoding errors; all 13 now parse after adding errors="surrogatepass" sanitization. Scanned-image-only PDFs remain unhandled — no OCR pipeline is wired in yet.
  • Page count shows 0 in the readiness report. The _extract_pdf result doesn't propagate page_count back through result.update() cleanly — a minor bug that was logged but does not affect word count or heading extraction.
  • Bucket scoring is name-pattern-first. The current scorer relies heavily on filename matching against 16 known WTE project names. It would classify a legitimately new WTE project incorrectly if its filename doesn't match. The second-best approach (used if name score is uncertain) falls back to word_count > 500 — a reasonable but imperfect heuristic.
  • Single-folder scope. The pipeline targets one Drive folder. Projects distributed across multiple folders or in Shared Drives (supportsAllDrives not yet enabled in the gws query) would be missed.
  • Local template PDF not yet ingested. The sample template/VCS_Soc Son_Project-Description.pdf was used to verify parsing feasibility but was not formally added to the corpus manifest. It should be copied into data/corpus/raw/verra/ before PHASE-02.
