Architecture

  • Provider abstraction: NoopProvider (zero cost, human-in-the-loop) is default
  • SQLite FTS5 with Porter stemmer — no external services
  • DraftSection: structured output with confidence, provenance, issues
  • DraftRun: persisted per-run JSON for traceability

Review gates

  • HIGH/CRITICAL sections get REVIEW REQUIRED markers if no corpus examples found
  • NoopProvider always returns UNSUPPORTED + REVIEW REQUIRED — forces human review
  • Cache ensures same section is not re-retrieved or re-drafted within a run
  • is_built() guard prevents crashes when index doesn't exist yet
06 / Limitations / Second-Best Alternative

Limitations / Second-Best Alternative

  • Index must be built before retrieval works — `RetrievalIndex.build()` must be called explicitly (via `pdd-agent build-index` CLI command). If called before building, `is_built()` returns False and all retrieval functions return []. The NoopProvider is a safe fallback but doesn't do real retrieval.
  • SQLite FTS5 on Windows holds file locks — On Windows, the WAL journal mode combined with exclusive connection mode caused PermissionError on cleanup in tests. Solved by using `idx.close()` in a `finally` block and removing the WAL pragma (not needed for single-writer access). Tests that previously called `idx.build()` with a temp dir now call `idx._open()` + `idx.is_built()` + `idx.close()` to avoid the lock issue.
  • Noembeddings or semantic search — BM25 keyword matching is used instead of dense embeddings. For WTE technical terminology this is adequate. If semantic search is later needed, a local embedding model (e.g. via Ollama) can be added as a new provider without changing the orchestration layer.
  • Win32 `isolation_mode` typo — The parameter is `isolation_level` not `isolation_mode`. This was a Python sqlite3 API misspelling in the original code.
  • No actual LLM integration yet — The NoopProvider is the only registered provider. API-backed providers (OpenAI, Ollama) are stubbed as importable modules but not yet functional. The pipeline is structurally complete; wiring a real provider is the PHASE-04/PHASE-05 concern.
07 / Errors / Warnings / Flags

Errors / Warnings / Flags Encountered

  • TypeError: isolation_mode — Python sqlite3 uses `isolation_level`, not `isolation_mode`. Fixed by removing the parameter entirely (Python 3.14 sqlite3 default is already WAL-with-exclusive for transactions when autocommit is off).
  • sqlite3.OperationalError: no such table: sections_fts — Drafting tests called `get_examples_for_section()` before `idx.build()` was called. Fixed by adding `index.is_built()` guards in `search.py`, `get_examples_for_section()`, and `get_section_heading_examples()`. Now gracefully returns [] instead of crashing.
  • PermissionError: WinError 32 on tempfile cleanup — Windows holds SQLite WAL locks after close() unless `conn.close()` is called AND the temp DB is closed before the directory cleanup. Fixed by adding explicit `idx.close()` in `finally` blocks in all test cases.
  • Assertion mismatch in _clean_query — Test expected `"landfill diversion"` (double space) but the cleaner normalizes all whitespace to single spaces: `"landfill diversion"`. Fixed test assertion to match the correct normalized behavior.
  • ModuleNotFoundError: pdd_agent.schemas — `section_orchestrator.py` imported `from pdd_agent.schemas.project_input` but the package is at root level `schemas/`. Fixed to `from schemas.project_input`.
  • Missing raw variable in get_examples_for_section — The `if sub_section_id: ... else: ...` branches didn't cover the case where neither branch assigned `raw`. The original code was an if without else: only the `if` branch assigned raw. Fixed by adding the `else: raw = index.get_section_examples(section_id, k=k)` branch.
  • Datetime deprecation warnings — `datetime.utcnow()` is deprecated in Python 3.12+. Used in `methodology_rules.py`, `section_orchestrator.py`, and `index.py`. Marked as warnings only; fix deferred.
