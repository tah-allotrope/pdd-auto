03 / Math / Algorithm Used

Math / Algorithm Used

The demo execution was primarily orchestration and validation, not numerical computation:

  • Schema validation: Pydantic ProjectInput.model_validate() on Inegol YAML.
  • Consistency checks: Cross-section numeric verification (net = baseline - project - leakage), crediting period total validation, capacity/waste throughput alignment.
  • TBD tracking: Regex scan across 36 sections for placeholder markers.
  • Comparison metrics: Countable comparisons (section count, table count, provenance entries, review flags, appendix count).

06 / Limitations / Second-Best Alternative

Limitations / Second-Best Alternative

  • Demo provider text is generic: The DemoProvider emits project-agnostic synthetic prose. Real Inegol-specific narrative depth would require an LLM provider with the actual corpus and intake data.
  • Quantification is TBD: Emission reduction numbers are null in the Inegol input, so quantitative sections contain placeholder logic rather than validated calculations.
  • Public participation appendix missing: The Codex output includes public participation meeting announcements (Appendix 1) which the pipeline does not yet generate.
  • Second-best alternative: Instead of using the generic DemoProvider, we could have integrated an OpenAI provider with the Inegol corpus for project-specific narrative generation. Rejected because it would require API keys and increase cost/complexity for a demonstration artifact.
07 / Errors / Warnings / Flags

Errors / Warnings / Flags Encountered

  • NoneType format error in _summarize_project (fixed): net_emissions_tco2e_per_year was None for Inegol. Fixed by adding conditional formatting with "TBD" fallback.
  • NoneType multiplication in _check_crediting_period_total (fixed): net * years crashed when net was None. Fixed by early return when any operand is None.
  • Section key format mismatch (fixed): section_map builder in consistency.py produced keys like "3.3.3" instead of "3.3", causing GHG boundary check to miss the section. Fixed by using sub_section_id directly when it already includes the parent prefix.
  • Demo text test failure (fixed): Updated test assertion to match new generic demo text.
  • No blockers: All 204 tests pass; Inegol demo runs with zero review flags.
