03 / Math / Algorithm Used

Math / Algorithm Used

Across all four phases, the core algorithmic work was validation, consistency checking, and structured rendering rather than heavy numerical computation:

  • Schema validation: Pydantic v2 with custom validators for net emissions, double-counting, coordinate bounds, crediting period years.
  • Consistency checking: Cross-section numeric verification (net = baseline - project - leakage), crediting period total validation, GHG boundary completeness (5 required elements), monitoring parameter coverage (4 ACM0022 params).
  • TBD detection: Regex scan with 5 specific marker types mapped to schema evidence requirements.
  • Table rendering: OOXML primitive composition (paragraphs, runs, shading, borders) via python-docx.

07 / Errors / Warnings / Flags

Errors / Warnings / Flags Encountered

  • NoneType multiplication (Phase 04): Fixed by early return when quantification values are None.
